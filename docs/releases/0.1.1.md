# pytest-warmup 0.1.1

First public release of `pytest-warmup`.

`pytest-warmup` is a pytest plugin for batch preparation and distribution of expensive test resources. It is aimed at test suites where resource creation is slow, order-sensitive, or requires explicit orchestration outside ordinary fixture wiring.

## Highlights

- Explicit `WarmupPlan.require(...)` declaration model.
- `@warmup_param(...)` injection for both fixtures and test functions.
- Narrow producer model built around `warmup_mgr.use(...).prepare(...)`.
- Optional producer conveniences through:
  - `producer_fixture="..."`
  - `warmup_autoresolve_producer`
- File-based snapshot overrides through `prepare(snapshot_file=...)`.
- Validation coverage for:
  - producer-chain errors
  - dependency cycles
  - distributed declarations
  - snapshot file failures
  - scope mismatch edge cases
- Public-package hygiene:
  - executable examples
  - built-wheel smoke testing
  - `twine check`
  - GitHub Actions CI
  - Trusted Publishing workflow for PyPI

## Included Examples

- `examples/basic_usage.py` for the default explicit producer path
- `examples/autoresolve_usage.py` for fixture-side `warmup_autoresolve_producer`
- `examples/named_producer_usage.py` for explicit producer disambiguation

## Status

This is still an early public release. The public API is intentionally narrow, but behavior and naming may still change as the package gets real-world feedback.

Current public surface:

- `WarmupPlan`
- `WarmupRequirement`
- `WarmupError`
- `@warmup_param(...)`
- `warmup_mgr.use(...).prepare(...)`

## Verification

Release artifacts are currently verified through:

- local test suite
- built-wheel smoke testing in a fresh virtual environment
- `twine check dist/*`

## Notes

- The package is intentionally not a general-purpose factory framework.
- The package does not try to hide producer orchestration behind autouse magic.
- Domain-specific plans remain outside the generic public package surface.
